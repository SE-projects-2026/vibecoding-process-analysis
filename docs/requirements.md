# 📋 요구사항 명세서 — FlashAI v2.0

**작성일:** 2026-06-04  
**버전:** v2.0  
**작성자:** 신주영 

---

## 1. 프로젝트 개요

텍스트를 붙여넣으면 Claude AI가 자동으로 Q&A 플래시카드를 생성해주는 웹 앱.  
v1.0 프로토타입을 바탕으로, 소프트웨어 공학 프로세스를 적용하여 품질을 개선한 버전.

---

## 2. v1.0 문제점 분석 (요구사항 도출 배경)

| 문제 | 영향 | 우선순위 |
|------|------|----------|
| 카드 수를 사용자가 조절 불가 (항상 4~6장) | 사용자 통제권 부족 | High |
| 입력 텍스트가 너무 짧아도 생성 시도 | 불필요한 API 호출 및 오류 | High |
| 카드별 학습 상태 추적 기능 없음 | 반복 학습 지원 불가 | Medium |
| API 오류 메시지가 불친절함 | 사용자 혼란 | Medium |
| 모바일 환경에서 카드 레이아웃 불안정 | 접근성 저하 | Low |

---

## 3. 기능 요구사항

### Must Have (핵심 기능)

| ID | 요구사항 | 검증 방법 |
|----|----------|-----------|
| F-01 | 사용자가 생성할 카드 수를 3 / 5 / 7장 중 선택할 수 있다 | UI에서 선택 후 정확한 수의 카드 생성 확인 |
| F-02 | 입력 텍스트가 30자 미만이면 생성 버튼 클릭 시 경고 메시지를 표시한다 | 짧은 텍스트 입력 후 동작 확인 |
| F-03 | 카드별 "알았어요 ✅ / 모르겠어요 ❌" 버튼으로 학습 상태를 표시할 수 있다 | 버튼 클릭 후 카드 상태 변화 확인 |
| F-04 | 학습 진도를 "X / Y장 완료" 형태로 상단에 표시한다 | 카드 상태 변경 시 카운터 업데이트 확인 |

### Should Have (권장 기능)

| ID | 요구사항 | 검증 방법 |
|----|----------|-----------|
| F-05 | API 오류 발생 시 원인에 따라 다른 안내 메시지를 표시한다 (네트워크 오류 / API 키 오류 등) | 오류 상황 시뮬레이션 |
| F-06 | 카드 생성 중 로딩 애니메이션(점 3개 깜빡임)을 표시한다 | 생성 버튼 클릭 후 시각적 피드백 확인 |

### Could Have (선택 기능)

| ID | 요구사항 | 검증 방법 |
|----|----------|-----------|
| F-07 | 생성된 카드를 텍스트 파일로 내보내기 할 수 있다 | 내보내기 버튼 클릭 후 파일 다운로드 확인 |

---

## 4. 비기능 요구사항

| ID | 요구사항 |
|----|----------|
| NF-01 | 단일 HTML 파일로 배포 가능해야 한다 (별도 서버 불필요) |
| NF-02 | 최신 Chrome, Firefox, Safari에서 정상 동작해야 한다 |
| NF-03 | 모바일(375px 이상) 화면에서 카드가 깨지지 않아야 한다 |
| NF-04 | Claude Sonnet 4 모델을 사용한다 |

---

## 5. 범위 외 (Out of Scope)

- 사용자 계정 및 로그인 기능
- 카드 저장 / 서버 동기화
- 다국어 지원
