## 1. 아키텍처 개요

단일 HTML 파일 구조 (HTML + CSS + JavaScript 인라인).  
외부 의존성: Claude API (`api.anthropic.com/v1/messages`)

```
[사용자 브라우저]
    │
    ├── UI Layer (HTML/CSS)
    │     ├── 입력 영역 (textarea + 카드 수 선택)
    │     ├── 카드 그리드 (flip 애니메이션)
    │     └── 진도 표시 바
    │
    └── Logic Layer (JavaScript)
          ├── generateCards()     — API 호출 및 카드 렌더링
          ├── renderCards()       — DOM 생성
          ├── updateProgress()    — 학습 진도 업데이트
          └── validateInput()     — 입력 검증
```

## 2. 주요 변경사항 (v1.0 → v2.0)

| 컴포넌트 | v1.0 | v2.0 |
|----------|------|------|
| 카드 수 | 고정 (4~6장) | 사용자 선택 (3 / 5 / 7) |
| 입력 검증 | 없음 | 30자 미만 경고 |
| 학습 상태 | 없음 | 카드별 ✅/❌ + 진도 표시 |
| 오류 처리 | 단순 메시지 | 오류 유형별 안내 |
| 로딩 | 텍스트만 | 점 3개 애니메이션 |

## 3. UI 컴포넌트 설계

### 3.1 입력 영역
```
[ textarea: 학습 내용 입력 ]
[ 카드 수: ○3장  ●5장  ○7장 ]   ← 라디오 버튼 (기본값: 5장)
```

## 4. API 호출 설계

**엔드포인트:** `POST https://api.anthropic.com/v1/messages`

**프롬프트 구조:**
```
다음 텍스트를 읽고 Q&A 플래시카드를 정확히 {count}개 만들어줘.
반드시 아래 JSON 형식만 반환하고 다른 텍스트는 절대 쓰지 마.
[{"q":"질문","a":"답변"}, ...]
텍스트: {userInput}
```

**오류 처리 분기:**
- `status 401` → "API 키를 확인해주세요"
- `status 429` → "요청이 너무 많습니다. 잠시 후 다시 시도해주세요"  
- `status 500+` → "서버 오류입니다. 잠시 후 다시 시도해주세요"
- `network error` → "네트워크 연결을 확인해주세요"
- `JSON parse error` → "응답 형식 오류입니다. 다시 시도해주세요"
